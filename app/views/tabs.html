Interaction principles

Only one tab can be active at a time.

The active tab should be distinguishable from the inactive, with maximum contrast ratio background:label.

Inactive tabs should have a hover state that is distinguisable with minimum contrast ratio background:label.

Focused tabs should be consistent with other focus states and distinguisable from active, inactive and hover states.

Horizontal rules can be used to separate groups of tabs.

Content principles

The label should reflect the thing you need do with the content of that tab, not a description of what it is.

Accessibility principles

The active tab should be labelled as active in the code.

Only active tab content should be readable.

User research

Syrian refugee resettlement found no problems with the pattern. Users have a good understanding of what to expect from the tabs

Application platform uses the pattern for long lists. When combined with a search, the pattern performs very well when users don't know the contents of the list.
