Error patterns

Let users access your service in a secure way.

  • Usage: internal/external
  • Status: experimental
  • Last updated: 05 March 2018

When a user accesses a web page, the server responds with a HTTP status code. In most cases a users jounrney isn't affected unless there is an error. Errors are sorted into two groups of statuses - 400 to 499 are client-side errors and 500 - 599 are server-side errors.

404 page

404 errors (sometimes known as 'page not found errors') tell a user the page they were trying to access is not there. Services should do the hard work so these errors don't happen.

Users encounter 404 errors when:

  • we make a mistake - for example a user follows a link that is broken
  • the user makes a mistake - inputting a link incorrectly

  • avoid listing the error code - a user shouldn't need to know the error code
  • make it clear how to get help
