Why have we created this pattern?

This has been seen in a few internal and external services and we need to identify its purpose: either navigating between content or filtering search results and standardise it for the relevant usage.

Interaction principles

  • try to make it so people don't need pagination either by setting the default number of results per page so that pagination is rarely needed (about 25-75 results per page).
  • identify what the sorting order should be: is it the most relevant/useful match at the top? What’s been recently modified?
  • provide large clickable hit areas and adequate spacing between pages
  • clearly identify current page
  • provide previous and next links, don’t rely on only arrows

Content principles

The content that opens the modal window (for example, anchor text) should explain to the user what will happen.

The content of the modal window should be focused on the single task and should not add unnecessary detail.

The content should be consistent with the referring page so that the user is not confused.

The modal window should have a clear main heading.

Accessibility principles

  • ensure there is sufficient colour contrast
  • avoid infinite scrolling as this causes issues for keyboard only users
  • font size is legible
  • provide large clickable hit areas and adequate spacing between pages
  • clearly identify current page
  • provide previous and next links, don’t rely on only arrows
