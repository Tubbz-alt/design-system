Contextual help

Small overlays containing help content for complex interactions.

  • Usage: internal
  • Status: experimental
  • Last updated: 24 September 2017
{% include "includes/patterns/help.html" %}

Why have we created this pattern?

The GDS pattern does not meet the user need.

Interaction principles

This pattern should only be used on complex internal services.

Help should be easy to close, move and minimise until the user needs it.

If contextual help doesn’t answer a user’s question, links to more comprehensive help should be easy to find.

Content principles

GDS has guidance for help content.

User research

  • users find side-by-side guidance more helpful and easier to use.
  • small overlay windows offer the smallest interference with the users flow
  • this is a good article on contextual help from Nielsen Norman
