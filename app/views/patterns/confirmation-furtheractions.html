Confirmation: More actions to complete
EXPERIMENTAL

This pattern is currently experimental because more research is needed to validate it.

Use this pattern when the online process is complete but you need to do something else to finish the application, such as send documents by post.

{% include "includes/components/conf--incomplete.html" %}

When to use this pattern

When a user needs to do something else, such as send documents, to complete an application offline. It should explain what the user needs to do, and what happens next in their application.

Services using this pattern

  • Sponsorship did the initial work but is now paused

Research

TThis variation of the confirmation pattern was created as research found that, with the standard GOV.UK confirmation pages, users did not realise that they still had to do something. The standard confirmation page was too final and seen as an end to what the user had to do.

There was a recurring theme with users doing screenshots of this page for their records, and wanting to know what had happened with their application. The link in the 'request received' box was included to reassure users they could still get access to their information.

In addition, the link to download the PDF doc was used as a backup, as the user was emailed their documents at the same time. It wasn’t essential that they used this interaction to download their document, but it was good to clarify what the next task was. It allowed them do it straight away, rather than relying on having to go to their emails.
