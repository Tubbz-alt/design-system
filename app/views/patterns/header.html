Service header

A familiar landmark that appears throughout the service at the top of every view.

  • Usage: internal
  • Status: recommended
  • Last updated: 05 August 2017
{% include "includes/patterns/header.html" %}

When to use this pattern

If your service isn’t on a service.gov.uk domain, use this header. Services not on a gov.uk domain cannot use the crown logo or new transport font.

How to use

The service name should link to the starting point of a service, usually the home page.

The header can be used for global actions, like signing out. But do not add more than two global actions, unless there is a proven need.

The header should be the name of the service. Find out more about naming your service.

User research

  • Too many global actions placed in the header impacted mobile/narrow view; minimise global actions on header.
  • Users didn't always associate service name with linking back to the service home page.
  • Long usernames on header were challenging; users preferred sign-out over username display as it offered reassurance they are leaving the system.
  • Service names were not verbs but nouns due to covering a range of actions.

Services using this pattern
