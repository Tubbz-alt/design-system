Service header

A familiar landmark that appears throughout the service at the top of every view.

  • Usage: internal
  • Status: recommended
  • Last updated: 05 August 2017
{% include "includes/patterns/header.html" %}

Interaction principles

The service name should link to the logical starting page; usually the home page of the service.

If users need to sign in to the service under different roles, ensure the starting page is appropriate for them.

The header can be used for global actions, for example, Search and Sign out.

Do not add more than two global actions, unless there is a proven need.

Content principles

The header should be the name of the service. Find out more about naming your service.

For internal systems, you should only use the ‘Digital by default’ logo if your service has a public-facing element on GOV.UK.

For internal systems, there is no need to use the GOV.UK blue bar.

User research

  • Too many global actions placed in the header impacted mobile/narrow view; minimise global actions on header where necessary
  • Users didn't always associate service name with linking back to the service home page
  • Long usernames on header were challenging; users preferred sign-out over username display as it offered reassurance they are leaving the system
  • Service names were not verbs but nouns due to covering a range of actions

  • Returns and movements
