Why have we created this pattern?

The GOV.UK header and footer pattern does not meet the user need, especially around internal services.

Interaction principles

The service name should link to the logical starting page; usually the home page of the service.

If users need to sign in to the service under different roles, ensure the starting page is appropriate for them.

The header can be used for global actions, for example, Search and Sign out.

Do not add more than two global actions, unless there is a proven need.

Content principles

The header should be the name of the service. Find out more about naming your service.

For internal systems, you should only use the ‘Digital by default’ logo if your service has a public-facing element on GOV.UK.

For internal systems, there is no need to use the GOV.UK blue bar.

Accessibility principles

User research

  • The service name was not immediately associated with linking to the home page of the service. Either a 'home' link had to be placed explicitly in the header or was factored as part of a breadcrumb trail.
  • Users, especially caseworkers, were very task-specific and required core functions to be placed in header along with global actions
  • Usernames deemed challenging in some cases due to character length (some being full email addresses); if account is tied to POISE, log-in from POISE would suffice and sign-out was preferred over username as it offered reassurance to users they are leaving the system
  • Header kept very minimal because of small singular monitors used to access internal services; where necessary, this may lead to capitalising on header area to place core nav and global items
  • Service names were not verbs but nouns as they cover a range of actions
