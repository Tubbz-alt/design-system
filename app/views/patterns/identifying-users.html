Authentication

Enable users to access your service and keep their data secure

  • Usage: internal/external
  • Status: experimental
  • Last updated: 18 February 2018

When to use authentication

Authentication enables users to verify who they say they are.

Most one-time services won't need to authenticate their users. When a service does need to verify a users identity, it needs to balance ease of access with keeping a service and user data secure.

You might want to authenticate users if:

  • your service allows users to save and complete a transaction at another time
  • a user can check the status of a transaction
  • a user will use a service multiple times

Authentication is not the same as identity assurance

Identity assurance confirms the true identity of a user. The checks are usually done before confirming a users suitability for something. Identity assurance can consist of:

  • confirming the user is a human
  • confirming the validaty of information and documents
  • fraud checks
  • checking for warnings about a user

User accounts

User accounts are a form of authentication. Avoid them where possible. Services should start in public so users can find the right service. This is valuable when there are hundreds of services like in Government.

Making users create accounts when not needed:

  • forces users to keep track of more logins then neccesary
  • creates a barrier for users and can reduce completion rates
  • burdens teams with databases that are difficult to build and maintain
  • grows the amount of redundant databases - which is expensive and a security risk

Only make users create an account if:

  • users access a service frequently
  • users need it - don't force users to create an account until necessary
  • a service needs to audit user behaviour

Guidance

Considerations you should make:

  • balance frequency of use, security requirements and any convenience benefits to the user
  • consider security trade-offs and data storage early in the design of your service
  • consult security specilists where needed
  • don’t use National Insurance numbers to verify a user’s identity
