Identifying users

Various forms of identifying and authenticating users

  • Usage: internal/external
  • Status: experimental
  • Last updated: 30 November 2017

Authenticating users

Authentication tells the system that a user who did something earlier is the same user. An example would be a user starting an application is the same user who returned to finish it later.

You can authenticate users using reference numbers, personal details, codes sent in SMS and emails or third-party authentication.

Authentication has limits. It does not:

  • confirm who the user is
  • what the user can do
  • if the user is who they say they are

Identity assurance

Identity assurance confirms the true identity of a user. The checks are usually done before confirming a users suitability for something. The assurance can consist of:

  • confirming the user is a human
  • confirming the validaty of information and documents
  • fraud checks
  • checking for warnings about a user

User accounts

Avoid user accounts where possible. Services should start in public so it is easier for users to find the right service. This is valuable when there are hundreds of services like in Government.

Making users create accounts when not needed:

  • forces users to keep track of more logins then neccesary
  • creates a barrier for users and can reduce completion rates
  • burdens teams with databases that are difficult to build and maintain
  • grows the amount of redundant databases - which is expensive and a security risk

Only make users create an account if:

  • users access a service frequently
  • users need it - don't force users to create an account until necessary
  • a service needs to audit user behaviour

Guidance

Considerations you should make:

  • balance frequency of use, security requirements and any convenience benefits to the user
  • consider security trade-offs and data storage early in the design of your service
  • consult security specilists where needed
  • don’t use National Insurance numbers to verify a user’s identity
