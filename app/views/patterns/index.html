Patterns

Patterns developed by the Home Office DDaT user research and design teams to help develop and create consistent, usable services. They include example components and guidance on usage.

Beta

  • Flash cards: interrupt the user flow with important information
  • Header: identify a common landmark at the top of every view

Alpha

  • Highlighting matches: helps users identify matching data
  • Modal: creates focus on a single interaction
  • Navigation: navigate to different at any time
  • Table multi-select: allow the user to select multiple rows by clicking anywhere on the row
  • Tabs: switch between views of grouped content or hide content that is not absolutely necessary to see
  • Timeline: time-ordered activity showing the history of a person or object with additional context

Discovery

  • Help: small overlays containing help content for complex interactions
  • Tasks: a list of completed and still-to-do tasks
