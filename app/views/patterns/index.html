Patterns

Patterns developed by the Home Office DDaT user research and design teams to help develop and create consistent, usable services. They include example components and guidance on usage.

Beta

  • Interruption card: an alert that interrupts the user flow with important information
  • Service header: a familiar landmark that appears throughout the service at the top of every view

Alpha

  • Highlighted search matches: highlighting helps users identify matching data
  • Modal window: creating focus on a single interaction
  • Subnavigation links: go to a different section at any time
  • Table multi-select: allow users to select all rows in a table and allow them to interact with the whole row rather than just checkbox
  • In-page tabs: switching between views of grouped content and hiding content that is not absolutely necessary to see
  • Vertical timeline: time-ordered activity showing the history of a person or object with additional context

Discovery

  • Contextual help: small overlays containing help content for complex interactions
  • Tasks: a list of completed and still-to-do tasks
