Patterns

Patterns developed by the Home Office DDaT user research and design teams to help develop and create consistent, usable services. They include example components and guidance on usage.

Beta

  • Interruption card: interrupt the user flow with important information
  • Service header: identify a common landmark at the top of every view

Alpha

  • Highlighted search matches: helps users identify matching data
  • Modal window: creates focus on a single interaction
  • Subnavigation links: navigate to different sections at any time
  • Table multi-select: allow the user to select multiple rows by clicking anywhere on the row
  • In-page tabs: switch between views of grouped content or hide content that doesn’t always need to be shown
  • Vertical timeline: time-ordered activity showing the history of a person or object with additional context

Discovery

  • Contextual help: small overlays containing help content for complex interactions
  • Task checklist: a list of completed and still-to-do tasks
