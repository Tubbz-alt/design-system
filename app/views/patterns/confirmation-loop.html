The confirmation pattern

What is a confirmation?

A confirmation is an interaction that asks the user to confirm before completing an action or transaction.

Confirmation may be needed where the action is final and a mistake is not rectifiable - leaving the user at a loss.

Guidance

It should be used:

  • to confirm a single input or action
  • when the action is not rectifiable
  • when the completion immediately starts a process

It should not be used:

  • to agree to terms and conditions
  • if the user can change their action/input later

The confirmation should be

  • meaningful
  • use the same language as the requested action
  • able to be cancelled and the input/action changed
