Why have we created this pattern?

A confirmation loop is a proposed solution to confirming a single piece of important information, for example, is this the right amount, is this the correct address, before allowing the user to complete.

Interaction principles

The field should have validation. Users shouldn't be asked to reinput the same information twice.

Only use the confirmation loop for a single input field. Only use the input when there is an immediate subsequent action, for instance the email will be sent straight away. Don't use if you are using task list or check your answers page.

User research

Research around interaction conducted, with no issues. No research on whether the process improves quality and accuracy of data input

Accessibility principles

Work has been done to make the confirmation loop accessible but it still relies on in page loading.
