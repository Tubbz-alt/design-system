Interruption card

An alert that interrupts the user flow with important information and must be acknowledged before continuing.

Interaction principles

An interruption card should be used to draw the attention of the user to some important information specific to that point in their journey.

Cards should have headings. You may want to hide the heading but it’s generally simpler not to.

One thing per card. If the user needs to do more than one thing, use extra cards.

Use larger text (24px) than in the rest of the journey.

Use a “Continue” button to advance to the next card or page.

Don’t use form elements (except buttons) within cards. Form elements will add extra complexity and you will need to deal with error states.

User research

Interruption cards have proved successful at getting users to read important information which they might skip if presented on a start page. This pattern has been so successful on the passport renewals service that we have been able to dramatically reduce the amount of content on the start page.

Interrupting users’ flow in this way can increase their understanding and retention of important information.

Accessibility principles

The colours used have been proved to be a better contrast for people with dyslexia
