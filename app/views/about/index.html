Home Office Design System

Design System for Home Office Digital, Data and Technology (DDaT). It extends the GOV.UK system with patterns and components for additional use cases and users. The intent is to help create a common design language amongst delivery teams, making it cheaper to build and scale products and services.

There is a Design System Working Group who are responsible for accepting new patterns. If you’ve got a question or suggestion share it on Slack #interaction-designers channel or in a GitHub issue.

How to use the design system

When designing services you should consider this framework:

  • all services - whether for public or internal users - must start by using GOV.UK design elements and patterns
  • if you find those don’t meet your users needs, refer to the Home Office Design system and the cross-Government Design Patterns wiki to see if the problem has been approached
  • if you need to design something new because there’s no existing pattern let us know on Slack #interaction-designers channel or in a GitHub issue
