About this resource

This is the Design Patterns for Home Office Digital, Data and Technology (DDaT). It extends the GOV.UK design patterns with patterns for additional use cases and users, particularly around internal services.

The intent is to create a common design language amongst delivery teams, making it cheaper to build and scale products and services.

There is a Design Patterns Working Group who are responsible for accepting new patterns, provide advice and recommendations to Interaction Designers on their use and maintaining the HO Design Patterns library.

How to use the design patterns

When designing services you should consider this framework:

  • all services - whether for the public or internal users - must start by using GOV.UK design elements and patterns
  • if you find those don’t meet your users needs, refer to Design Patterns Hackpad and the Home Office Design Patterns to see if your problem has been solved there
  • if that doesn’t work raise this in the Interaction Designers slack channel or via the cross government design community - it’s highly likely someone else has solved your particular problem.
    • if you need to design something from scratch because there’s no existing pattern - that’s ok. If you think that it should be a pattern then follow the contribution guidance.

Please note that this resource will be used by our Service Assessment Team to check that it is being adhered to.
