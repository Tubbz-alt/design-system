When to use these components

Secondary buttons might be needed on admin interfaces with more then one thing per page.

How to use these components

Buttons need to have a good level of affordance and look like buttons.

Buttons can have an icon to further support their action or state.

Button text should describe the action the button will perform. Be particularly careful when labelling "Cancel" buttons so that the action is clear.

User research

Users sometimes confuse grey buttons with disabled buttons.

Users sometimes confuse white buttons with input fields.

White buttons have performed well in confirmation pages and interruption cards.
