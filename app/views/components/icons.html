Why have we created this pattern?

Expanding on GOV.UK guidance for icons to include interactions that allow users to interrogate information. This is more relevant to internal users.

Types of icons

The icons used here are grouped as follows:

  • Navigation icons – help users navigate through a service
  • Controller icons – help users to sort, organise or adjust the information, for example by filtering data, opening and closing modals or revealing additional content
  • Quick reference icons – help users to understand options and context

Interaction principles

  • The minimum size for an icon is 16px
  • If it's part of a button the hit area should be 36px
  • The icon shouldn't change colour in a hover/active/focus states, only the background.
  • In the normal state, the icon can be a different colour using the GOV.UK colour palette - both in solid and reversed out state - as long as the colour combination meets the minimum WCAG 2.0 AA colour contrast check

Content principles

  • Every icon should use a visible and audible label to avoid confusion over what it represents.

  • Align the icon in a central position in relation to any type
  • Type can sit either to the left or to the right of the icon

Accessibility principles

  • icons should always have a label and use aria-hidden="true"
  • if an icon can't have a label, ensure it has screenreader only support

<span class="visuallyhidden">Home button</span>

User research

  • users interpret icons differently so shouldn't be used without a label
  • don't use generic icons for specific labels, for example the calendar icon for a date of birth

Pattern is missing

  • more access needs research including cognition and comprehension
  • more research using icons as :before content on buttons
  • greater knowledge of what icons are used within icon libraries (don't use them all)
  • feedback on designer/developer needs
  • clarity on alignment, padding and size for touchscreen hit area
