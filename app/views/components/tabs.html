Interaction principles

Only one tab can be active at a time.

The active tab should be distinguishable from the inactive, with maximum contrast ratio.

Inactive tabs should have a hover state that is distinguisable with minimum contrast ratio.

Focused tabs should be consistent with other focus states and distinguisable from active, inactive and hover states.

Inclusive components has some really good guidance on tabs if you want to read more.

Content principles

The label should reflect the thing you need do with the content of that tab, not a description of what it is.

Accessibility principles

The active tab should be labelled as active in the code.

Only active tab content should be readable.

User research

Syrian refugee resettlement found no problems with tabs. Users have a good understanding of what to expect from them.

Dyslexic users were ignoring the tabs. So the line around the tabs were made thicker and the blue links were underlined to avoid this issue.

Returns and movements found users missed the inactive tab. They think that might be due to content but its worth keeping an eye on.
